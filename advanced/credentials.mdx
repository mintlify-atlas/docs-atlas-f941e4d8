To automate tasks across services like GitHub, Slack, or Google Sheets, Automatisch needs to store the credentials that authorize it to act on your behalf. This page explains how those credentials are protected, how to manage connections from the UI, and what happens when a credential expires.

## How credentials are stored

Automatisch never stores your credentials in plain text. Before writing any credential to the database, it encrypts it using **AES (Advanced Encryption Standard)** — a U.S. Federal Information Processing Standard (FIPS) that is widely used to protect sensitive data worldwide.

The encryption key is the `ENCRYPTION_KEY` environment variable you set during deployment. Automatisch uses this key both to encrypt credentials when they are saved and to decrypt them when a flow needs to use them.

<Warning>
**Never change `ENCRYPTION_KEY` after your instance is running.** If you rotate this key, Automatisch can no longer decrypt any previously stored credentials. Every existing connection will fail, and you will need to reconnect every app from scratch. Set a strong, random value once during initial setup and treat it as a long-lived secret.
</Warning>

To generate a strong encryption key:

```bash
openssl rand -hex 32
```

Set the output as the value of `ENCRYPTION_KEY` in your `docker-compose.yml` for both the `main` and `worker` services.

## Adding an app connection

<Steps>
  <Step title="Open app connections">
    In the Automatisch sidebar, navigate to **App Connections**. This page lists all apps you have already connected and provides a button to add new ones.
  </Step>
  <Step title="Choose an app">
    Click **Add connection** and select the app you want to connect from the list of available integrations.
  </Step>
  <Step title="Enter your credentials">
    Fill in the credentials form for the selected app. Depending on the app, this may be an OAuth flow (where you are redirected to the third-party service to authorize access) or a form asking for an API key or similar credential.
  </Step>
  <Step title="Save and verify">
    After completing the authorization, Automatisch saves the encrypted credential and displays the connection in your list. You can click **Test** on any connection to confirm it is working.
  </Step>
</Steps>

---

## Testing a connection

After adding a connection, or if you suspect it has stopped working, you can test it at any time:

1. Go to **App Connections** in the sidebar.
2. Find the connection you want to verify.
3. Click the **Test** button next to it.

Automatisch will attempt to make an authenticated request to the third-party service using the stored credential and report whether it succeeded.

## When credentials expire

Many third-party services issue access tokens that expire after a set period. When this happens, flows that depend on that connection will fail. Automatisch surfaces this as a connection error in the flow execution log.

<Note>
For apps that support OAuth refresh tokens, Automatisch automatically refreshes the access token in the background. You do not need to reconnect these apps manually unless the refresh token itself is revoked.
</Note>

If a connection has expired and cannot be automatically refreshed:

<Steps>
  <Step title="Identify the affected connection">
    Check the execution log for the failed flow. The error message will indicate which app connection failed.
  </Step>
  <Step title="Reconnect the app">
    Go to **App Connections**, find the affected connection, and click **Reconnect** (or delete the connection and add it again). Complete the authorization flow to issue a fresh credential.
  </Step>
  <Step title="Re-enable the flow">
    Once the connection is restored, return to the flow and verify it runs successfully.
  </Step>
</Steps>

---

## Deleting a connection

Deleting a connection removes the stored credential from the database permanently. Any flows that use that connection will stop working.

<Warning>
Before deleting a connection, check whether any active flows depend on it. Removing the credential will cause those flows to fail immediately.
</Warning>

To delete a connection, go to **App Connections**, find the entry you want to remove, and click **Delete**.
