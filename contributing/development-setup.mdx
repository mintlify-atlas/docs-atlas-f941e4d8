---
title: "Development setup"

---

This guide walks you through running Automatisch locally for development. You'll need to start the backend server, a background worker, and the frontend separately.

## Prerequisites

- **Node.js** (check `.node-version` or `.nvmrc` in the repo for the required version)
- **PostgreSQL 14** — other versions may cause issues with the provided database scripts
- **Redis** — used by the job queue (BullMQ)
- **Yarn** — the repo uses Yarn as its package manager

<Warning>
  The database scripts included with Automatisch are tested against **PostgreSQL
  14**. If you have a different version installed, you may encounter errors
  during database setup.
</Warning>

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the `main` branch from GitHub:

    ```bash
    git clone git@github.com:automatisch/automatisch.git
    cd automatisch
    ```

  </Step>

  <Step title="Install dependencies">
    Install dependencies for the backend and frontend packages separately:

    ```bash
    # Backend dependencies
    cd packages/backend
    yarn install

    # Frontend dependencies
    cd ../web
    yarn install
    ```

  </Step>

  <Step title="Configure the backend">
    Copy the example environment file and edit it as needed:

    ```bash
    cd packages/backend
    cp .env-example .env
    ```

    The defaults in `.env-example` are suitable for local development. You can review the full list of configuration options in [`packages/backend/src/config/app.js`](https://github.com/automatisch/automatisch/blob/main/packages/backend/src/config/app.js).

  </Step>

  <Step title="Set up the database">
    Make sure PostgreSQL and Redis are running, then create the database, run migrations, and seed an initial user:

    ```bash
    cd packages/backend

    # Create the development database
    yarn db:create

    # Run all migrations
    yarn db:migrate

    # Create the seed user (user@automatisch.io / sample)
    yarn db:seed:user
    ```

    <Warning>
      `yarn db:create` expects a `postgres` superuser to exist. If you don't
      have one, either create a superuser named `postgres` manually, or create
      the database yourself using the connection values from
      [`packages/backend/src/config/app.js`](https://github.com/automatisch/automatisch/blob/main/packages/backend/src/config/app.js).
    </Warning>

  </Step>

  <Step title="Start the backend server">
    Run the API server in one terminal:

    ```bash
    cd packages/backend
    yarn dev
    ```

    The backend listens on `http://localhost:3000` by default.

  </Step>

  <Step title="Start the worker">
    In a second terminal, start the background job worker:

    ```bash
    cd packages/backend
    yarn worker
    ```

    The worker processes queued jobs such as flow executions and webhook deliveries.

  </Step>

  <Step title="Start the frontend">
    Copy the example environment file and start the Vite dev server in a third terminal:

    ```bash
    cd packages/web
    cp .env-example .env
    yarn dev
    ```

    The frontend opens automatically at [http://localhost:3001](http://localhost:3001).

  </Step>
</Steps>

## Logging in

Use the seed credentials to log in for the first time:

| Field    | Value                  |
| -------- | ---------------------- |
| Email    | `user@automatisch.io`  |
| Password | `sample`               |

<Note>
  Change your email and password in the account settings before using your
  local instance for anything sensitive.
</Note>

## Running the docs site

If you're working on documentation, you can run the docs server locally:

```bash
cd packages/docs
yarn install
yarn dev
```

The docs site is available at [http://localhost:3002](http://localhost:3002).
