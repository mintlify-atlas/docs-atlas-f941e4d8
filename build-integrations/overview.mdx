Automatisch integrations connect third-party services to the platform so users can include them in their flows. If an app you need is not in the [available apps](/guide/available-apps) list, you can build it yourself.

This section is written for developers who want to extend Automatisch with new integrations. You should be comfortable with JavaScript and have a working [development setup](/contributing/development-setup) before you start.

<Note>
  The terms **integration** and **app** are used interchangeably throughout this section.
</Note>

## What is an integration?

An integration is a Node.js module that lives in `packages/backend/src/apps/`. Each integration encapsulates everything Automatisch needs to work with a third-party service:

- **Auth** — how to authenticate users and store their credentials
- **Triggers** — how to detect new events in the service and pass data into a flow
- **Actions** — how to perform operations in the service using data from earlier steps

Each part of the integration receives the `$` global variable, which provides an authenticated HTTP client, access to stored credentials, step parameters configured by the user, and methods to emit output data. See [Global variable ($)](/build-integrations/global-variable) for full details.

## Tech stack

Integrations are written in Node.js using ES module syntax (`import`/`export`). No bundler or compilation step is required — files are loaded directly by the backend at runtime.

The HTTP client bundled in `$` is a pre-configured [Axios](https://axios-http.com) instance. It sets `apiBaseUrl` as the base URL and applies any `beforeRequest` hooks you define on the app (for example, to attach an authorization header on every request).
